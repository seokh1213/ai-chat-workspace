# Okinawa AI Trip Planner Design

트리플 여행 일정 화면을 기준으로 한 지도 중심 설계다. 핵심은 “장소를 찾고, 일정에 넣고, AI와 동선을 고치는 과정”이 한 화면에서 끊기지 않는 것이다.

## Product Intent

- 사용자는 지도를 보면서 여행 날짜별 동선을 판단한다.
- AI 채팅은 보조 패널이 아니라 일정 편집 도구다. 장소 추가, 순서 변경, 메모 보강, 우천 대안 제안을 바로 반영해야 한다.
- 좌표는 내부 데이터 품질이며 사용자에게 통계로 노출하지 않는다. 지도 핀과 동선으로 자연스럽게 드러나면 충분하다.
- 조사 장소와 일정은 서로 계속 오가야 하므로 별도 페이지로 분리하지 않는다.

## Layout Decision

### Desktop

- 전체는 `왼쪽 작업 사이드바 + 중앙 지도 + 오른쪽 AI 상담` 구조다.
- 지도는 항상 보이는 1순위 화면이다.
- 왼쪽 작업 사이드바에는 `일정`과 `조사 장소` 섹션을 세로로 쌓는다.
- `일정`과 `조사 장소`는 각각 접고 펼칠 수 있다.
- 왼쪽 작업 사이드바 자체도 접을 수 있다.
- 오른쪽 AI 상담 패널도 접을 수 있다.
- 왼쪽과 오른쪽 패널을 모두 접으면 지도만 크게 남는다.
- 접힌 패널은 지도 위 작은 오버레이 버튼으로 다시 연다.

### Mobile / Narrow

- 상단은 지도, 하단은 작업 사이드바를 세로 스택으로 둔다.
- 트리플처럼 지도 높이는 과하게 크지 않게 유지하고, 작업 사이드바에서 날짜/장소를 빠르게 넘긴다.
- 사용자가 지도를 크게 보고 싶을 때는 지도 영역을 확장할 수 있어야 한다.

## Navigation Model

- `일정` 섹션: Day 탭, 일정 노드, 일정 편집, 순서 조정, 삭제, 지도 포커스.
- `조사 장소` 섹션: 검색, 장소 카드, 현재 Day에 추가, 지도 포커스.
- AI 채팅: 자연어로 장소 추가/삭제/순서 변경/동선 상담.
- 지도 핀 클릭: 해당 위치에 걸린 일정과 조사 장소를 모두 보여준다.

## Visual Language

트리플의 여행 앱 느낌은 밝은 배경, 얇은 구분선, 선명한 포인트 색, 작은 카드, 지도 위 오버레이에서 나온다. 마케팅 페이지처럼 보이면 안 된다.

### Colors

- Page background: `#F5F6F8`
- Surface: `#FFFFFF`
- Primary text: `#24262D`
- Secondary text: `#5F6673`
- Muted text: `#9AA0A6`
- Hairline: `#EAEDF0`
- Strong line: `#DDE2E7`
- Triple teal: `#1FC1B6`
- Triple violet: `#975FFF`
- Restaurant coral: `#FF6168`
- Hotel mint: `#26CEC2`
- Product orange: `#FF8224`
- Route line: `#1FC1B6`

### Color Roles

- `#1FC1B6`: primary action, selected Day route, online AI status.
- `#975FFF`: 모든 일정 핀. 선택한 Day는 진하게, 다른 Day는 낮은 opacity로 표시한다.
- `#6B7280`: 조사 장소 핀. 일정 핀보다 낮은 우선순위로 보이게 한다.
- `#FF6168`, `#26CEC2`, `#FF8224`: 이후 카테고리별 핀을 세분화할 때 사용한다.

## Typography

- System UI stack을 사용한다.
- Letter spacing은 `0`으로 둔다.
- 패널 내부 제목은 14-16px, 본문은 12-13px 중심으로 조밀하게 설계한다.
- 큰 히어로 타입은 쓰지 않는다. 이 앱은 반복해서 쓰는 작업 도구다.

## Map

- 현재 구현은 Leaflet과 OpenStreetMap 타일을 사용한다. 트리플 export는 Google Maps 기반이지만, UX만 가져오고 지도 공급자는 유지한다.
- 모든 Day의 일정 핀은 보라 계열로 표시한다.
- 선택한 Day의 좌표가 있는 일정 노드는 순서대로 선으로 연결한다.
- 선택한 Day의 각 일정 노드는 번호 마커를 가진다.
- 리스트에서 지도 보기 버튼을 누르면 해당 핀으로 이동하고 핀이 짧게 강조된다.
- 같은 위치에 여러 일정 또는 장소가 있으면 숫자 배지는 쓰지 않고 팝업에서만 항목을 함께 보여준다.
- 범례는 지도 위 작은 흰색 카드로 둔다. route, 일정, 조사 장소만 설명한다.

## Schedule Nodes

- 일정 노드는 시간, 제목, 분류, 메모, 액션 버튼을 가진다.
- 긴 메모는 처음에는 4줄 프리뷰로 보여주고 `전체 보기`로 확장한다.
- 편집 모드에서는 시간, 분류, 제목, 메모를 한 카드 안에서 수정한다.
- 노드 순서를 바꾸면 지도 동선 선도 즉시 다시 그려진다.

## Places

- 조사 장소는 “데이터 검수 목록”이 아니라 “일정에 넣을 후보 목록”으로 보여야 한다.
- 별점, 리뷰, 카테고리, 메모를 짧게 보여준다.
- 각 장소의 주 액션은 `현재 Day에 추가`와 `지도에서 보기`다.
- 좌표 여부, 조사 개수 같은 내부 상태는 기본 UI에서 숨긴다.

## AI Chat

- 오른쪽 채팅은 작업 중 계속 열려 있는 보조 조작면이다.
- 사용자의 메시지는 teal 배경, AI 메시지는 흰 카드로 구분한다.
- placeholder는 명령 예시를 짧게 제공한다.
- AI 응답이 실제 일정 변경을 만들었으면 리스트와 지도에 즉시 반영한다.

## Components

- Cards: white surface, 1px hairline, 8px radius 이하.
- Buttons: 4px radius, 아이콘 중심. 텍스트 버튼은 명확한 명령에만 사용한다.
- Day tabs: 작은 segment 버튼. active 상태는 teal 배경.
- Sidebar sections: `일정`, `조사 장소` 두 개를 세로로 쌓고 각각 collapse를 제공한다.
- Popups: 지도 핀에 걸린 항목을 리스트로 보여준다.
- Resizers: 패널 경계에 얇은 핸들을 둔다. hover/focus 시 teal로 표시한다.

## Interaction Rules

- 지도는 패널 조작 때문에 사라지면 안 된다.
- 사용자가 조사 장소를 일정에 추가하면 왼쪽 사이드바와 `일정` 섹션을 자동으로 펼친다.
- Day를 바꾸면 지도 동선도 선택 Day 기준으로 즉시 변경한다.
- 핀 색만으로 의미를 강요하지 않고 범례와 팝업 라벨을 함께 제공한다.
- 카운터성 문구보다 실제 행동 버튼을 우선한다.

## Implementation Notes

- 현재 지도 공급자: Leaflet + OpenStreetMap.
- 트리플에서 가져올 UX 패턴: 날짜별 route polyline, 번호 마커, active marker 강조, 지도 위 범례/오버레이, 지도 중심 작업 흐름.
- 유지할 앱 고유 기능: AI 채팅, 자연어 일정 편집, 로컬 저장, URL 기반 지도 viewport 복원.
